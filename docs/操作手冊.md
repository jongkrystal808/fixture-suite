# Fixture-M 治具管理系統操作手冊

版本：v1.0（依目前程式碼與畫面整理）  
適用對象：現場操作員、工程師、系統管理員

## 1. 系統目的與範圍
Fixture-M 用於管理治具全生命週期，包含：
- 治具主資料（治具、站點、機種、負責人）
- 收料 / 退料交易（序號、Datecode）
- 使用記錄與更換記錄
- 庫存與壽命分析
- 治具與機種關聯查詢

## 2. 角色與權限
- `admin`：可使用資訊管理功能（治具/站點/機種/負責人等主資料維護）
- 一般使用者：可登入、查詢、登錄交易與記錄（依 API 權限與客戶權限設定）

注意：系統採「客戶情境」操作，登入後必須有可用 `customer`，且每次 API 都會帶 `X-Customer-Id`。

## 3. 登入前準備
### 3.1 連線網址
- 前端入口（Nginx）：`http://localhost/`
- API 文件（Swagger）：`http://localhost/docs`
- FastAPI 健康檢查：`http://localhost:8000/api/health`

### 3.2 首次環境啟動（Docker）
在專案根目錄執行：
```powershell
docker compose up --build
```

啟動後服務預設：
- MySQL：`localhost:3307`
- FastAPI：`http://localhost:8000`
- Nginx：`http://localhost`

### 3.3 帳號資料
目前初始化 SQL 以建表為主，未提供固定預設登入帳密。  
若無法登入，請由管理員先建立使用者資料，並設定該使用者可用的客戶範圍。

## 4. 基本操作流程（建議順序）
建議先完成主資料，再做日常交易：
1. 建立站點
2. 建立機種
3. 建立治具
4. 建立機種與站點/治具需求關聯
5. 進行收料
6. 記錄使用
7. 記錄更換（報廢/維修）
8. 在「庫存與壽命」追蹤結果

## 6. 日常操作
### 6.1 登入與客戶切換
1. 開啟系統首頁。
2. 輸入帳號與密碼登入。
3. 若有多客戶權限，從頂部下拉切換客戶。
4. 切換客戶後，畫面資料會依新客戶重新載入。

### 6.2 操作中心：記錄收料/退料
路徑：`操作中心 -> 記錄收料/退料`

可建立三種記錄型態：
- `batch`：序號區間（起始/結尾）
- `individual`：序號清單（逗號分隔）
- `datecode`：Datecode + 數量

共同必填重點：
- 交易類型：`receipt`（收料）或 `return`（退料）
- 治具 ID
- 記錄類型
- 來源類型：`customer_supplied`（客供）或 `self_purchased`（自購）

序號輸入建議：
- 純數字序號請補零為固定寬度（例如 `15 -> 0015`）
- 批量序號前綴需一致，且結尾不可小於起始

可用功能：
- 新增單筆
- 匯入 Excel
- 下載匯入範本
- 查看最近 10 筆
- 在「收/退料總檢視」做多條件查詢

### 6.3 操作中心：記錄治具使用
路徑：`操作中心 -> 記錄治具使用`

核心欄位：
- `fixture_id`、`model_id`、`station_id`
- `use_count`（必須 > 0）
- 記錄層級：`individual` 或 `batch`（序號型）

規則：
- Datecode（`lifecycle_mode=fixture`）治具不可寫入 usage。
- `uses` 壽命模型僅允許序號層級使用記錄。

可用功能：
- 單筆輸入
- Excel 匯入（可下載範本）
- 依機種預覽使用量（不寫入）
- 依機種批次寫入使用記錄

### 6.4 操作中心：記錄治具更換
路徑：`操作中心 -> 記錄治具更換`

支援：
- `record_level=serial`（單序號）
- `record_level=fixture`（整體數量）

事件類型：
- `scrap`（報廢）
- `maintenance`（維修）

關鍵規則：
- `serial` 模式必填 `serial_number`
- `fixture` 模式必填 `scrap_qty` 且 > 0
- 若治具屬 Datecode 模式，`fixture` 更換時必填 `datecode`

可用功能：
- 單筆輸入
- Excel 匯入（可下載範本）

### 6.5 庫存與壽命
路徑：`庫存與壽命`

主要用途：
- 在庫總覽：查看在庫、部署、維修、報廢等狀態
- 壽命分析：查看 normal / warning / expired 狀態
- 快速追蹤即將更換治具

### 6.6 查詢中心
路徑：`查詢中心`

可進行：
- 治具查詢
- 機種查詢
- 治具與機種關聯反查（由治具找機種、由機種找治具）
- 點選治具/機種開啟明細抽屜

### 6.7 資訊管理（Admin）
路徑：`資訊管理`

可維護主資料：
- 治具管理
- 站點管理
- 機種管理（含站點綁定、治具需求）
- 負責人管理

注意：此區通常需要 `admin` 權限。

## 7. 匯入/匯出操作建議
- 先下載範本，再填資料，避免欄位不符。
- 匯入前刪除示例列，保留欄位名稱。
- 序號欄位建議全以文字處理，避免 Excel 自動轉數字。
- 出現部分失敗時，先看錯誤列號與訊息，再修正重匯。

## 8. 常見錯誤與排除
- `尚未選擇客戶`：先在頂部選擇客戶。
- `401 Token 無效或過期`：重新登入。
- `需要管理員權限`：請使用 admin 帳號。
- `治具不存在或不屬於此客戶`：確認治具 ID 與當前客戶。
- `datecode 模式需提供 datecode 與 quantity > 0`：補齊欄位。
- `batch 模式需填 serial_start / serial_end`：補齊區間。
- `序號不可用`（SP 回傳）：確認序號是否已退料/報廢/非在庫。

## 9. 日結建議流程（現場）
1. 完成當日收退料登錄。
2. 補齊當日使用記錄。
3. 若有異常，登錄更換（報廢/維修）。
4. 到「庫存與壽命」確認警示清單。
5. 匯出必要報表備查。
